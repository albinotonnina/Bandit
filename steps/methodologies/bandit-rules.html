Rules of Bandit

Use variables for everything

                    
$spacing: 10px;
$spacing-half: ($spacing/ 2);

$color-brand-1: #363a3e;
$grey-mist: #f5f5f5;
$grey-platinum: #bbb;

$width-nav: 250px;

$zindex-modal: 1400;

$font-normal: 400;

Magic numbers and errant values are banned

A key selector in only two files:
the theme file and the layout file

MyKeySelector-layout.scss 
                    
.MyKeySelector{
    @include your-grid-generator-row();
}

.MyKeySelector-items{
    @include your-grid-generator-column(9);
}

.MyKeySelector-list{
    @include your-grid-generator-column(3);
}
MyKeySelector-theme.scss 
                    
.MyKeySelector-items{
     color: $color-red-20;
}

.MyKeySelector-list{
    padding-left: $default-padding;
}

Duplicate code

Abstract only when it really makes sense

Really, you can do a lot with 1kb of well written CSS

CSS stats for Facebook

No Nesting

NO 
                    
.MyKeySelector{}

.MyKeySelector .MyKeySelector-items{}
YES 
                    
.MyKeySelector{}

.MyKeySelector-items{}
                    
                
                    
/* works well with SASS too */

.MyKeySelector{
    &-items{}
}

Override key selectors where you define them

MyKeySelector-theme.css 
                    
.MyKeySelector{
    color: #ffcc00;
}

.myoverride .MyKeySelector{
    color:blue;
}
MyKeySelector-theme.scss 
                    
.MyKeySelector{
    color: $color-red-20;

    .myoverride &{
       $color-blue;
    }

}

No ID selectors

Too specific

No type selectors

Use class names as the right-most selector

  • The HTML structure is not tightly coupled to the styles
  • Poor performance (selectors are evaluated right to left)

Avoid extending selectors

MyKeySelector-theme.scss 
                    
.MyKeySelector{
    @extend .generic-class;
}

It saves little and it gives a bad DX

Use Mixins

Don't abuse mixins

Be responsive

Class names that express sizes don't work well with breakpoints

Use Grids

Use bootstrap grid mixins, do your own, but use grids

Think Mobile First

Even if you don't need to

Use tools for the grunt work

let autoprefixer add the vendor specific styles

let postcss convert your inline assets to encoded strings (and many other things)
