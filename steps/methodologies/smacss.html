SMACSS

Scalable and Modular Architecture for CSS

SMACSS in a nutshell

Split styles into categories

  1. Base for defaults and resets
  2. Layout for page sections
  3. Module for modular reusable elements
  4. States for element variations
  5. Theme is like states but for the look of modules and layouts

SMACSS in a nutshell

Merits

  • Good organizational system
  • Increases semantics
  • Decreases reliance on specific HTML
  • More style guide, less rigid framework

SMACSS in a nutshell

Faults

  • Difficult to qualify styles, what category?
  • Hard to maintain adherence to those definitions
